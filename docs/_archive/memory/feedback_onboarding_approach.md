Always research and document before touching code. User wants step-by-step alignment on roteiro before implementation.

**Why:** User values research-backed decisions and wants to be involved in every design decision. Previous onboarding docs were created without enough user input.

**How to apply:**

- Never start coding onboarding without approved roteiro
- No auto-scroll in previews — let the professional scroll themselves
- No skip buttons on narrative screens (user wants to force reading, but open to discussing)
- Walkthrough = narrative text screens with Motion, NOT tooltip/spotlight overlays
- Upload UX: modern file picker ("toque aqui" → opens file browser → shows preview)
- Crop tool for profile photos — professional chooses exactly what to show
- Minimal data per screen — never many fields on same screen
- Cinematographic Motion transitions, but purposeful (not decorative)
- PIX = recorrente automático (like iFood), not copia-e-cola

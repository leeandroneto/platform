Don't try to pre-define exact playbooks for all pages upfront. Build each page spec one at a time, with the user providing opinions and new ideas as we advance.

**Why:** The user wants creative control over each page. Pre-defining everything removes their ability to shape the direction as they learn from each completed page. They want a general strategy but with room for iteration.

**How to apply:** Have a high-level plan (order, shared components, architecture) but leave individual page specs to be filled collaboratively, one page at a time. The user will bring references, screenshots, and new ideas for each page.

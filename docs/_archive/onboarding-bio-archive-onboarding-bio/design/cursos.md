1. Apple — Human Interface Guidelines (HIG)
   Este é o padrão mais alto do mercado. Se você seguir isso, seu app parecerá "caro". Eles ensinam exatamente como organizar camadas e profundidade.

Onde encontrar: developer.apple.com/design/human-interface-guidelines

O que focar: Procure pelas seções de "Layout", "Navigation" e "Modality". A Apple explica perfeitamente quando usar uma folha que sobe (Sheet) e quando trocar de tela.

Vídeos (Altíssimo Nível): Procure por "WWDC Design Sessions" no app/site Apple Developer. São aulas gratuitas dos designers da Apple.

2. Google — Material Design 3 (Material You)
   O Google é mestre em densidade de informação. O Material Design 3 é focado em como adaptar o layout para qualquer tela (mobile até desktop).

Onde encontrar: m3.material.io

O que focar: Seção de "Layout" (entenda o sistema de grids) e "Components". Eles explicam a diferença entre Bottom App Bar, Navigation Rail e Navigation Drawer.

Curso Google: No site Google Design, há estudos de caso reais sobre como apps organizam informações complexas.

3. Meta (React Native) — Design System Principles
   Embora o React Native seja uma tecnologia, a Meta foca muito em padrões de performance percebida.

Referência: Procure pelo blog de engenharia/design da Meta.

O que aprender: Como lidar com listas infinitas e carregamento de dados (o segredo de apps como Instagram e Facebook).

4. Cursos de "Elite" (Não Oficiais, mas Padrão de Mercado)
   Se você quer aprender a "estética SaaS moderna" (estilo Linear/Stripe), estes são os melhores:

Refactoring UI: Criado pelos fundadores do Tailwind CSS. Não é um curso de mobile, mas é o melhor guia do mundo sobre hierarquia visual e densidade. Ele ensina exatamente como colocar 400 info em uma tela sem parecer bagunça.

Shift Nudge (Matt D. Smith): Considerado o melhor curso de UI Design do mercado para quem quer chegar no nível de "Interface Designer" profissional.

Erik Kennedy (Learn UI Design): Excelente para entender lógica de cores e espaçamento.

O Plano de Estudo para o seu problema (Configurações Mobile)
Se o seu mobile está virando bagunça com 5 informações, seu problema não é código, é Arquitetura de Informação. Recomendo ler especificamente sobre estes 3 conceitos:

Progressive Disclosure (Divulgação Progressiva): Não mostre tudo. Mostre apenas o que o usuário precisa agora. O iFood esconde "detalhes do pedido" dentro de um clique, ele não deixa tudo na home.

Visual Anchors (Âncoras Visuais): O olho precisa de um lugar para descansar. Se tudo tem a mesma cor e tamanho, o cérebro "buga". Use pesos de fonte diferentes.

The Box Model for Mobile: Pare de pensar em linhas. Pense em Containers.
